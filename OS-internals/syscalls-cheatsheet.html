System Calls

The controlled gateway between user space and the kernel
Ring 3 · User Trap · Transition Ring 0 · Kernel
🚪The Boundarywhy syscalls exist
User Space — Ring 3 — restricted
Your Process
Runs application code. Cannot touch hardware, other processes' memory, or privileged CPU instructions directly.
libc Wrapper
Functions like read() / write() are thin wrappers that set up registers and issue the trap.
⚡ Trap Instruction · privilege switch
Kernel Space — Ring 0 — privileged
Syscall Handler
Validates arguments, looks up the number in the syscall table, dispatches to the real kernel function.
Hardware / Drivers
The kernel — and only the kernel — talks to disks, NICs, memory-management units, and the scheduler.
Core idea: user code is deliberately powerless. A syscall is the single, audited doorway a process uses to ask the kernel to do something privileged on its behalf — the kernel checks every request before acting.
⚙️The Mechanismwhat actually happens on x86-64
🔁 Trap → Dispatch → Return
1
Set up registers. Syscall number goes in rax; args in rdi, rsi, rdx, r10, r8, r9.
2
Issue the trap. The syscall instruction switches CPU from Ring 3 to Ring 0 and jumps to a fixed kernel entry point (LSTAR MSR).
3
Kernel runs. Handler validates args, indexes the syscall table by rax, calls the implementation (e.g. sys_read).
4
Return. Result (or -errno) is placed in rax; sysret drops back to Ring 3 and user code resumes.
📇 Registers & Entry (x86-64 Linux)
# write(1, buf, len) by hand mov rax, 1 ; __NR_write mov rdi, 1 ; fd = stdout mov rsi, buf ; *buf mov rdx, len ; count syscall ; trap → Ring 0 ; rax now = bytes written ; or -errno on failure
Legacy: 32-bit used int 0x80; modern 64-bit uses the faster syscall/sysret pair. The vDSO serves some calls (like gettimeofday) entirely in user space to skip the trap.
💸Why They're Expensivethe cost of crossing the line
🔀 Mode Switch
Cost~100–400 ns per round trip — hundreds of cycles even with nothing else.
WhatCPU flushes pipeline, swaps stacks (user → kernel), saves/restores registers.
NoteA mode switch is not a context switch — same process, no scheduler involved.
🛡️ Mitigations Tax
Spectre/MeltdownKPTI, retpolines and barriers added real overhead to every kernel entry.
CacheKernel code/data evict some of your process's cache lines on entry.
📉 The Fix: Batch
Bufferstdio buffers writes so one write() flushes many printf()s.
Scatter/gatherreadv/writev move multiple buffers per call.
io_uringsubmit many I/O ops with near-zero syscalls via shared ring buffers.
Interview trap: "Is a system call a function call?" — No. A function call is a jump within your address space. A syscall is a privilege-level transition via a trap instruction; the CPU changes rings and control lands at a fixed kernel entry point, not at an address you chose.

⚠️ EINTR
A blocking syscall can be interrupted by a signal and return -EINTR. Robust code retries the call. This surprises people who assume read() always completes or fails cleanly.
🔒 seccomp
Kernel filter that whitelists which syscalls a process may make. The basis of container/sandbox hardening — a blocked call gets the process killed or an errno.
Interview trap: "What's the difference between an exception, an interrupt, and a system call?" All three trap into the kernel, but: a syscall is synchronous and voluntary (software asks); an exception is synchronous and involuntary (page fault, div-by-zero); an interrupt is asynchronous and hardware-driven (NIC, timer).
