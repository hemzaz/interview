① Auth & Validation
kubectl runs local schema validation before sending
API Server runs admission chain: authn → authz → mutating webhooks → validating webhooks
etcd stores desired state only after admission passes
Response is 202 Accepted — not 200 OK — because reconciliation is async
resourceVersion in etcd prevents concurrent write conflicts
② Control Plane
Scheduler watches API Server via informer cache — not etcd directly
Scoring phases: Filter (removes unfit nodes) → Score (ranks survivors)
Filter plugins: NodeAffinity, Taints/Tolerations, ResourceFit, PodTopologySpread
Score plugins: LeastAllocated, ImageLocality, InterPodAffinity
Binding writes spec.nodeName to API Server (updates etcd)
③ Node Ops
Kubelet watches its own node's pods via informer — triggered by nodeName binding
Validates: securityContext, volume mounts, resource limits
Calls containerd CRI gRPC → containerd → containerd-shim → runc
Image pull uses imagePullSecrets → kubelet authenticates to registry
Sets up cgroups (limits) + CNI (networking) + CSI (volumes)
④ Readiness
livenessProbe failure → container restart (does not affect Endpoints)
readinessProbe failure → removed from EndpointSlice → no traffic
startupProbe disables liveness/readiness until startup succeeds
Pod status Running ≠ pod Ready — readiness gates determine traffic
minReadySeconds on Deployment: pod must be ready for N seconds before counted
202 ≠ deployed. kubectl apply returning success means the API Server accepted the manifest. The pod may not exist yet — reconciliation is async. Use kubectl rollout status to wait.
Scheduler watches API Server, not etcd. All control plane components communicate through API Server only. Direct etcd access is reserved for API Server itself. Controllers use informer caches.
Image pull happens on the node. If your cluster nodes can't reach the registry (air-gapped, no egress), image pull fails with ErrImagePull → ImagePullBackOff. Pre-pull images or use a mirror.
Running ≠ Ready. A pod in Running phase with a failing readinessProbe has Ready: False and is not in the Service EndpointSlice. Traffic is not sent to it.
kubelet watches its node's pods only. kubelet doesn't see pods assigned to other nodes. Each kubelet watches a filtered list: spec.nodeName=<itself>.
OOMKill is not ImagePullBackOff. Both restart containers. OOMKill exit code = 137. ImagePullBackOff stays in Waiting state — never reaches Running. Check kubectl describe pod → Events.
