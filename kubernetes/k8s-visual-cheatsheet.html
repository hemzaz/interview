Component Deep Dives // key internals
⚡
kube-apiserver
Single front-door · Stateless · :6443
RoleOnly component that reads/writes etcd. Everything else talks to it.
HA modeStateless — run N replicas behind a load balancer. No leader election needed.
WatchHTTP/2 chunked streaming. etcd → apiserver → all watchers (fan-out pattern).
AdmissionMutating → defaulting → Validating. Order is enforced by apiserver.
Interview trap: kube-proxy, kubelet, controllers do NOT talk to etcd. They watch/update via apiserver only.
🗃️
etcd
Raft consensus · Key-value · Source of truth
Key format/registry/pods/default/my-pod → protobuf
Raft writesLeader replicates to followers. Commit after majority ack. ⌊N/2⌋+1 quorum.
LinearizableReads always reflect committed state. Enables optimistic concurrency via resourceVersion.
Sizing3 or 5 members. Fast SSDs only. Often the #1 cluster bottleneck at scale.
Compaction: etcd grows unbounded without compaction. Set --auto-compaction-retention.
🎯
kube-scheduler
Filter → Score → Bind · Lease HA
TriggerWatches Pods with spec.nodeName == "". That's it — unbound pods only.
FilteringRemove infeasible nodes: resources, taints, nodeSelector, affinity, PV topology, Pod anti-affinity.
ScoringRank remaining nodes. Top scorer wins. Writes Binding subresource (not a full update).
Does NOTStart containers. Only writes nodeName. kubelet does the rest after seeing the binding.
Scheduling framework: Plugin extension points at PreFilter, Filter, PostFilter, Score, Bind. Replaces old predicates/priorities.
🔄
kube-controller-manager
Single binary · Shared informers · Per-binary leader election
PatternWatch → enqueue key → dequeue → reconcile → API write. Idempotent by design.
Shared factoryAll controllers for same resource type share one informer cache. One Watch per resource to apiserver.
HA gotchaLeader election is per binary, not per controller. One active instance runs ALL controllers.
Dep → PodsDeployment ctrl → creates ReplicaSet. ReplicaSet ctrl → creates Pods. Two-step chain!
Interview trap: Does Deployment controller create Pods? No — it creates/updates ReplicaSets. ReplicaSet ctrl makes Pods.
🖥️
kubelet
Node agent · CRI gRPC · :10250
TriggerWatches Pods where spec.nodeName == thisNode. Sees binding written by scheduler.
CRI stackkubelet → CRI gRPC → containerd → OCI runtime (runc) → namespaces + cgroups
Pause containerInfra container holds the network namespace. App containers join it via PID 1 model.
CNICalled after container create. Sets up veth pair, assigns Pod IP (Cilium/Calico/Flannel).
Probes: Liveness (restart), Readiness (remove from endpoints), Startup (delay liveness). All run on kubelet.
🌐
Service Networking
kube-proxy · ClusterIP · iptables / IPVS
ClusterIPVirtual IP — exists only in iptables/IPVS rules, not on any real interface. Pure kernel construct.
FlowEndpointSlice ctrl (kcm) → watches Pods+Services → kube-proxy watches EndpointSlices → programs DNAT.
IPVS modeKernel load balancer. O(1) rule lookup vs O(n) iptables chains. Better at 10k+ Services.
Cilium eBPFReplaces kube-proxy. Socket-level DNAT via eBPF maps. Lower latency, no iptables.
DNS: CoreDNS resolves svc.ns.svc.cluster.local → ClusterIP. Lives in kube-system, watched by kubelet.

Pod QoS Classes
Eviction order under node memory pressure
BestEffort No requests/limits set
Evicted FIRST OOM score: 1000
Burstable Requests < Limits (or partial)
Evicted second OOM score: 2–999
Guaranteed requests == limits for ALL containers
Evicted LAST OOM score: <10
CPU throttling vs OOM: CPU over-limit → throttled (cgroups). Memory over-limit → OOM killed by kernel.

Common Interview Gotchas
Things that trip people up
finalizers
Deletion Blocked?
Object gets deletionTimestamp but won't delete until controller removes all finalizers from the array.
ownerRef
Cascade Delete
GC traverses ownerReference graph. Foreground: wait for dependents. Background: owner deleted first.
CrashLoop
Backoff Resets
kubelet exponential backoff: 10s→20s→40s→… max 5m. Resets after 10m of stable running.
resourceVer
Optimistic Lock
Write with stale resourceVersion → 409 Conflict. Re-read, get fresh version, retry. etcd linearizability.
SSA
Server-Side Apply
apiserver tracks field managers. Conflict if two managers own same field. Use --force-conflicts carefully.
informer
Cache Sync
Controllers must call WaitForCacheSync() on start. Acting on stale cache = incorrect reconcile.
Reconciliation Pattern // all controllers follow this
🔁
The Control Loop
Desired state ← → Actual state
// Every Kubernetes controller is this loop func (r *Reconciler) Reconcile(ctx, req) (Result, error) { // 1. Read from LOCAL INFORMER CACHE (not API server) obj := r.cache.Get(req.NamespacedName) if obj == nil { return Done } // 2. Compute desired state (pure business logic) desired := computeDesired(obj) // 3. Compare with actual (may call API for owned objects) actual := r.getActualState(ctx, obj) // 4. Apply the diff via API server (may trigger more events) if diff(actual, desired) != nil { r.applyDesired(ctx, desired) // write via apiserver } // 5. Return: requeue after X, or Done return ctrl.Result{RequeueAfter: 30s}, nil }
Idempotency is mandatory. Reconcile may run 10× for one event (retries, requeues, restarts). It must always produce the same result for the same inputs.

Debug Commands
First things to run in an interview scenario
# Always start here — events tell the story kubectl describe pod <name> | tail -30 # Events sorted by time (namespace-wide) kubectl get events --sort-by='.lastTimestamp' -A # Previous crash container logs kubectl logs <pod> --previous -c <container> # Check what controller sees (Dep → RS → Pods) kubectl get deploy,rs,pods -l app=<label> -o wide # Node allocatable vs requested kubectl describe node <n> | grep -A8 Allocated # Force-clear stuck namespace finalizers kubectl patch ns <ns> \ -p '{"metadata":{"finalizers":[]}}' \ --type=merge # Dry-run to check admission without writing kubectl apply -f manifest.yaml --dry-run=server
Tip: --dry-run=server runs the full admission pipeline without writing to etcd. Great for debugging webhook issues.
