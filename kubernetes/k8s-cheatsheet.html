Kubernetes Interview Cheatsheet

Control Plane · Internals · Runtime
Control Plane Architecture
kube-apiserver
RoleSingle front-door for all cluster state. Every component talks to it — nothing talks directly to etcd except apiserver.
ProtocolREST + gRPC (internal). Watches via HTTP/2 chunked streaming.
Auth pipelineAuthn → Authz (RBAC/ABAC/Webhook) → Admission → Validation → etcd write
AdmissionMutating webhooks first, then Validating webhooks. Built-ins: LimitRanger, ResourceQuota, PodSecurity.
HAStateless — can run N replicas behind a load balancer.
Key interview point: apiserver is the only etcd client. All other components watch API objects, not etcd directly.
etcd
RoleSole source of truth for all cluster state. Distributed key-value store (Raft consensus).
Data modelKeys like /registry/pods/default/my-pod → protobuf-encoded object
Watch mechanismetcd streams change events to apiserver, which then fans out to all watchers (controllers, kubelet, etc.)
Raft quorumNeed (N/2)+1 members to accept writes. Typical: 3 or 5 nodes.
Performanceetcd is the most common bottleneck. Keep it on fast SSDs, separate from worker nodes.
Raft in brief: Leader receives writes, replicates to followers, commit only after majority ack. Leader election on failure.
kube-scheduler
RoleWatches for unscheduled pods (spec.nodeName == ""), picks a node, writes spec.nodeName back.
Two phases1) Filtering — remove infeasible nodes (resource, taints, nodeSelector, affinity, PV topology). 2) Scoring — rank remaining nodes, pick highest.
Scheduler does NOTstart containers. It only writes nodeName. kubelet on that node sees the binding and does the rest.
ExtensibilityScheduling Framework: plugins per extension point (PreFilter, Filter, Score, Bind, etc.)
HAMultiple replicas — leader election via lease. Only leader schedules.
kube-controller-manager
RoleSingle binary hosting all core reconciliation loops. Each controller watches API objects and drives actual → desired state.
Key controllersNode, ReplicaSet, Deployment, StatefulSet, DaemonSet, Job, CronJob, Endpoints/EndpointSlices, ServiceAccount, Namespace, GC, ResourceQuota
PatternWatch → enqueue → reconcile → API write. Uses shared informer factory (one cache per resource kind, all controllers share it).
HALeader election is per-binary, not per-controller. One active instance runs all controllers.
Trick question: Does Deployment controller create Pods? No — it creates/updates ReplicaSets; ReplicaSet controller creates Pods.
cloud-controller-manager
RoleRuns cloud-specific controllers split out from kube-controller-manager. Allows K8s core to remain cloud-agnostic.
ContainsNode (cloud node lifecycle), Route (cloud routing tables), Service (cloud load balancers)
Not presentOn bare-metal clusters or if you handle cloud resources differently (e.g., Cluster API).
kubelet (node agent — not control plane but critical)
RoleRuns on every node. Watches for Pods scheduled to its node, drives container lifecycle via CRI.
CRITalks to container runtime (containerd, CRI-O) via gRPC. Runtime calls OCI-compliant runc/crun.
ReconcilesPod spec → containers running, probes passing, volumes mounted, status reported back to apiserver.
Does NOTUse Docker directly (deprecated). Talks CRI only. kube-proxy handles networking rules — kubelet handles pod lifecycle.
What Happens When You kubectl apply a Manifest
Full Event Chain
1
kubectl computes the patch
kubectl reads local YAML, serializes it, computes a strategic merge patch against the last-applied annotation. Sends HTTP PATCH (or PUT for --overwrite) to kube-apiserver.
2
Authn + Authz
apiserver authenticates the request (cert, token, OIDC). Then RBAC/Webhook authorization checks if the principal has create/update verb on that resource/namespace.
3
Mutating Admission Webhooks
Sent to all registered MutatingAdmissionWebhooks in parallel. Can modify the object (inject sidecars, set defaults, add labels). Result merged back into the object.
4
Schema Validation + Defaulting
Object validated against OpenAPI schema. Built-in admission plugins run (e.g., LimitRanger injects resource defaults, PodSecurity checks policy). Object is now fully defaulted.
5
Validating Admission Webhooks
All ValidatingAdmissionWebhooks run (in parallel). Can only reject, not modify. Policy engines (OPA/Gatekeeper, Kyverno) hook here. Any rejection aborts the request.
6
etcd Write
apiserver serializes to protobuf, writes to etcd. etcd replicates via Raft. Once majority commits, apiserver returns 200 OK (update) or 201 Created. Object now has a resourceVersion.
7
Watch event fans out
apiserver broadcasts a MODIFIED/ADDED watch event to all watchers. Informer caches in controller-manager, scheduler, and kubelet update. Relevant controllers enqueue a reconcile.
8
Controller reconciles
e.g., Deployment controller sees new Deployment → creates/updates ReplicaSet → ReplicaSet controller creates Pod objects with spec.nodeName: "". Each write re-triggers the watch chain.
9
Scheduler binds Pod → Node
Scheduler sees unbound Pod, runs Filter + Score, writes nodeName back via a Binding subresource. kubelet on that node detects the Pod, calls CRI to pull image + start containers. Status flows back via kubelet → apiserver → etcd.
Admission Plugin Order
1 Authn → 2 Authz → 3 Mutating → 4 Schema → 5 Validating
Critical: Mutating runs before Validating. A mutating webhook can inject a field; a validating webhook then checks it. Order matters for policy correctness.
Informer / Watch Internals
ListWatchOn start, informer does a full List to populate cache, then opens a Watch stream for deltas.
Shared cacheAll controllers for the same resource kind share one informer cache — only one Watch to apiserver per resource type per binary.
Work queueInformer enqueues object key (namespace/name) on event. Controller dequeues, re-fetches fresh object from cache, reconciles.
resourceVersionPassed to Watch calls for resumption after disconnect without full re-list.
Strategic Merge Patch vs Others
Strategic MergeK8s default for apply. Merges lists by key (e.g., containers by name). Won't blow away existing fields not in your YAML.
JSON Merge PatchRFC 7396. Replaces entire list if you provide one. Used by some tooling.
JSON PatchRFC 6902. Explicit add/remove/replace operations on specific paths.
Server-Side ApplySince 1.18 GA. apiserver owns field management, tracks who set each field. Use --server-side.
Leader Election Mechanics
ResourceUses a Lease object (kube-system namespace). Replicas race to update the holderIdentity field.
AcquireCAS (compare-and-swap) on Lease via resourceVersion. Only one writer wins — etcd's linearizability guarantees this.
Renew intervalLeader renews every renewTime. If lease expires (leaseDuration), others race to acquire it.
kcm vs schedulerEach binary has its own separate Lease. Leader election is per-binary, not per-controller.
Node Components + Networking Plane
Container Runtime Interface (CRI)
Stackkubelet → CRI gRPC → containerd/CRI-O → OCI → runc/crun → Linux namespaces + cgroups
containerdDefault runtime. Manages image pull, snapshot, container create/start/stop via shim.
Pause containerEach Pod has an infra/pause container that holds the network namespace. App containers join it. Dies last.
CNICalled by kubelet after container create. Sets up Pod network (assigns IP, veth pair, routes). Examples: Cilium, Calico, Flannel.
kube-proxy + Service Networking
Modesiptables (default): chains per Service/Endpoint. IPVS: kernel load-balancer, better at scale. nftables: newer replacement for iptables.
ClusterIPVirtual IP — only exists in iptables/IPVS rules, not on any interface. kube-proxy programs DNAT rules to real Pod IPs.
EndpointSlice ctrlkube-controller-manager watches Pods + Services, maintains EndpointSlice objects. kube-proxy watches those.
Cilium eBPFCan replace kube-proxy entirely. Programs eBPF maps in kernel for low-latency socket-level load balancing.
Pod Scheduling Constraints
nodeSelectorHard match on node labels. Simple equality filter.
Node AffinityrequiredDuringScheduling = hard; preferredDuringScheduling = soft scored. Supports set operators.
Taints/TolerationsTaint on node repels Pods. Toleration on Pod allows scheduling. Effects: NoSchedule, PreferNoSchedule, NoExecute.
Pod Affinity/Anti-AffSchedule relative to other Pods' topology keys. Anti-affinity spreads replicas across zones.
TopologySpreadConstraintMax skew across topology domains. Replaces manual anti-affinity for spread.
Common Interview Questions & Gotchas
Reconciliation Loop Pattern
All controllers follow this exact pattern:
// Simplified reconcile loop func Reconcile(ctx, req) { obj := cache.Get(req.NamespacedName) // read from local cache desired := computeDesired(obj) // business logic actual := getCurrentState(ctx) // may hit API if diff(actual, desired) != nil { applyDiff(ctx, desired) // API write } return requeue // or Done }
Idempotency: reconcile must be safe to run multiple times. Never assume it runs exactly once.
Resource Versions & Optimistic Concurrency
resourceVersionAn opaque etcd revision string on every object. Changes on every write.
Optimistic lockIf you PUT/PATCH with a resourceVersion, apiserver rejects with 409 Conflict if object changed since you read it.
Controller patternControllers re-read from cache before writing to get fresh resourceVersion, then retry on 409.
Common Gotchas
FinalizersObject with finalizers won't be deleted — only gets deletionTimestamp set. Controller must remove finalizer to complete deletion.
OwnerReferencesGC uses these to cascade delete. Foreground deletion waits for dependents; Background deletes owner first.
Pod evictionNode pressure → kubelet evicts pods by QoS class: BestEffort first, Burstable next, Guaranteed last.
CrashLoopBackOffKubelet applies exponential backoff (10s → 20s → 40s … max 5m) before restarting a crashed container.
ImagePullBackOffCRI failed to pull image. Separate from crash — check image name, tag, registry credentials (imagePullSecrets).
QoS Classes
GuaranteedAll containers have requests == limits for CPU and memory. Never OOM-killed first. Lowest eviction priority.
BurstableAt least one container has requests or limits set, but not all equal. Middle tier.
BestEffortNo requests or limits set on any container. First to be evicted under node pressure.
Tip: Set requests = limits for critical Pods to guarantee Guaranteed QoS. But over-provisioning hurts bin packing — balance carefully.
Key kubectl Debugging Commands
# Check events (often most useful first) kubectl describe pod <name> # Real-time pod logs with previous crash kubectl logs <pod> --previous -f # Dump all events sorted by time kubectl get events --sort-by='.lastTimestamp' # Check what a controller sees kubectl get replicaset,pods -l app=<label> # Node conditions + allocatable resources kubectl describe node <node> | grep -A5 Conditions # Force delete stuck namespace (remove finalizers) kubectl patch ns <ns> -p '{"metadata":{"finalizers":[]}}' --type=merge
